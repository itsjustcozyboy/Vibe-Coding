# Product Requirements Document (PRD)

## 1. Overview

| 항목 | 내용 |
|------|------|
| 제품명 | Todo API |
| 버전 | 1.0.0 |
| 작성일 | 2026-04-07 |
| 상태 | In Development |

### 1.1 목적

팀원 모두가 공유하여 사용할 수 있는 **Todo 리스트 백엔드 API**를 제공한다.  
각 팀원은 자신의 계정으로 로그인하여 개인 할 일을 생성·관리하고, 팀 내에서 동일한 서버를 함께 사용한다.

### 1.2 배경 및 문제 정의

- 팀원들이 별도의 도구 없이 간단하게 할 일을 관리할 공용 백엔드가 필요하다.
- 기존 SaaS 툴은 팀 플랜 비용이 발생하거나 외부 의존성이 생기는 문제가 있다.
- 직접 서버를 운영하여 데이터를 완전히 통제하고자 한다.

## 4. 기능 요구사항 (Functional Requirements)

### 4.1 인증 (Authentication)

- **FR-01** 이메일(고유) + 비밀번호(최소 6자)로 회원가입할 수 있어야 한다.
- **FR-02** 로그인 성공 시 JWT(만료 7일)를 발급해야 한다.
- **FR-03** 인증이 필요한 모든 엔드포인트는 `Authorization: Bearer <token>` 헤더를 요구해야 한다.
- **FR-04** 비밀번호는 bcrypt(salt rounds ≥ 10)로 해시 저장해야 한다.

### 4.2 Todo 관리

- **FR-05** Todo는 제목(필수), 설명, 우선순위(low/medium/high), 마감일을 가진다.
- **FR-06** 사용자는 자신의 Todo만 조회·수정·삭제할 수 있다.
- **FR-07** 목록 조회 시 `completed`, `priority` 파라미터로 필터링할 수 있다.
- **FR-08** 목록 조회 시 `sortBy`(created_at, updated_at, due_date, priority), `order`(ASC/DESC)로 정렬할 수 있다.
- **FR-09** 목록 조회 시 `limit`(기본 50), `offset`으로 페이지네이션을 지원한다.
- **FR-10** Todo 수정 시 `updated_at`이 자동 갱신되어야 한다.

## 5. 비기능 요구사항 (Non-Functional Requirements)

| 항목 | 요구사항 |
|------|----------|
| 보안 | 비밀번호 bcrypt 해시, JWT 서명 검증, 타 사용자 데이터 접근 차단 |
| 입력 검증 | 모든 요청 입력에 대해 서버 측 유효성 검사 수행 |
| 배포 | Docker Compose 한 명령으로 실행 가능 |
| 확장성 | 향후 팀 규모 증가에 대비해 PostgreSQL 사용 |
| 테스트 | 핵심 흐름(인증, Todo CRUD)에 대한 통합 테스트 포함 |

## 6. API 엔드포인트 요약

| Method | Path | 인증 | 설명 |
|--------|------|------|------|
| POST | `/auth/register` | 불필요 | 회원가입 |
| POST | `/auth/login` | 불필요 | 로그인 |
| GET | `/users/me` | 필요 | 내 프로필 조회 |
| GET | `/todos` | 필요 | Todo 목록 조회 |
| POST | `/todos` | 필요 | Todo 생성 |
| GET | `/todos/:id` | 필요 | Todo 단건 조회 |
| PATCH | `/todos/:id` | 필요 | Todo 수정 |
| DELETE | `/todos/:id` | 필요 | Todo 삭제 |
| GET | `/health` | 불필요 | 서버 상태 확인 |

## 7. 성공 지표 (Success Metrics)

- 팀원 전원이 공용 서버에 계정을 생성하고 API를 사용할 수 있다.
- `docker-compose up` 단일 명령으로 서버가 정상 기동된다.
- 통합 테스트 전체 통과 (`npm test`).
- 잘못된 JWT로 보호 엔드포인트 접근 시 401 응답이 반환된다.
